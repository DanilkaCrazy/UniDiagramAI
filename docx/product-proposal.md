1. Описание
================================================================================
UniDiagramAI — это веб-приложение, интеллектуальный ассистент на базе
мультиагентных LLM, который по текстовому описанию на естественном языке
создаёт диаграммы различных типов (UML, BPMN, IDEF0, маркетинговые воронки,
архитектурные схемы и др.). Пользователь описывает процесс или структуру,
а система генерирует интерактивную диаграмму, которую можно сразу
редактировать в графическом редакторе. Никакого кода, никакого drag-and-drop.

================================================================================
2. Почему это нужно? (Проблема и боль)
================================================================================
Создание диаграмм — критически важная, но трудоёмкая задача.

| Тип решения                    | Ограничения                                           |
|--------------------------------|-------------------------------------------------------|
| Ручные редакторы (Visio, Draw) | Отнимают часы, отвлекают от содержания на оформление.|
| Специализированные инструменты | Сложны, дороги, требуют длительного обучения.        |
| Текстовые генераторы (Mermaid) | Требуют знания кода, неудобны для быстрых итераций.  |
| AI-помощники (существующие)    | Ограничены одной нотацией или не понимают русский язык.|

Пользователь вынужден выбирать между скоростью (но с кодом) и удобством
(но с ручным рисованием).

4. Решение: что делает PoC
================================================================================
PoC (MVP за 4 недели) реализует базовый сценарий:

1. Чат-интерфейс: ввод запроса на русском/английском.
2. Мультиагентная обработка:
   • Оркестратор определяет тип диаграммы.
   • Агент структурирования преобразует текст в JSON (Pydantic-схема).
   • Агент валидации проверяет логику и соответствие нотации.
   • Агент визуализации рендерит диаграмму.
3. Интерактивный редактор: перетаскивание, изменение свойств, экспорт.
4. Уточнение через чат: «Сделай этот блок синим», «Добавь шаг...».
5. Сохранение состояния диалога (memory).

5. Ключевые отличия от аналогов
================================================================================
• Понимание русского языка — основной фокус.
• Мультиагентная архитектура — высокое качество генерации.
• Поддержка широкого спектра нотаций (UML, BPMN, воронки, в будущем IDEF0).
• Интерактивность + AI — сочетание мощного редактора и уточнений через чат.
• Отсутствие промежуточного кода — пользователь работает только с текстом.

6. Метрики успеха
================================================================================
Продуктовые метрики:
  • Среднее время создания диаграммы: ≤3 мин (цель)
  • Доля успешных генераций без ручных правок: ≥80%
  • Пользовательская удовлетворённость (SUS): ≥4.5 из 5

Агентские метрики:
  • Точность распознавания нотации (F1): ≥0.85
  • Доля запросов, прошедших валидацию с первой попытки: ≥70%
  • Корректность интерпретации связей: (качественная оценка)

Технические метрики:
  • p95 latency: ≤10 сек
  • Стоимость одной генерации: ≤$0.05 (OpenAI GPT-4 или аналоги)

7. Сценарии использования
================================================================================
Основной сценарий:
  Пользователь вводит: «Нарисуй диаграмму последовательности для входа в
  систему: пользователь вводит логин/пароль, система проверяет, если ок —
  открывает сессию, иначе выдаёт ошибку».
  → Система генерирует UML Sequence диаграмму.
  → Пользователь уточняет через чат: «Переименуй "Успех" в "Открыть сессию"».
  → Экспортирует диаграмму в PNG.

Edge-кейсы:
  • Неполный запрос: «Нарисуй процесс заказа» → агент уточнения задаёт
    встречный вопрос о типе диаграммы.
  • Логическая ошибка: отсутствует начальное событие в BPMN → валидатор
    возвращает ошибку, система просит уточнить.
  • Синхронизация правок: пользователь вручную добавил элемент, затем через
    чат просит изменить его цвет — система находит элемент и применяет
    изменение без потери ручных правок.
