## UniDiagramAI 
Это интеллектуальный ассистент на базе мультиагентных LLM, который по текстовому описанию на естественном языке (русском или английском) создаёт диаграммы любых типов: UML (классов, последовательностей, прецедентов и тд..), BPMN, IDEF0, архитектурные схемы, маркетинговые воронки и другие. Никакого кода, никакого drag-and-drop — просто опишите, что нужно, и получите готовую диаграмму, которую можно тут же отредактировать в интерактивном графическом редакторе.

## Задача, целевая аудитория и текущая боль
### Для кого?

- Бизнес-аналитики, системные аналитики, архитекторы ПО
- Проектировщики, методологи, специалисты по качеству
- Маркетологи, product-менеджеры (воронки, CJM)
- Инженеры, строители (сетевые графики, IDEF0)
- Все, кто регулярно создаёт визуальные модели

### Какая боль сегодня?

- Ручное рисование в редакторах (Visio, Draw.io, Miro) отнимает часы, требует расстановки элементов, выравнивания, подбора нотации.
- Специализированные инструменты (Business Studio, Visual Paradigm) сложны, дороги и требуют длительного обучения.
- Текстовые генераторы (PlantUML, Mermaid) заставляют писать код на специальных языках — это неудобно для аналитиков без технического бэкграунда.
- Нет единого инструмента, который понимает русский язык и умеет строить диаграммы любых типов в одной экосистеме.
- AI-функции в существующих продуктах либо отсутствуют, либо реализованы как примитивные шаблоны, не способные к полноценной генерации.

Результат: низкая скорость документирования, высокий порог входа для новых сотрудников, разрозненные инструменты.

## Что сделает PoC на демо (минимум жизнеспособный продукт)
### PoC будет реализован за 4 недели и продемонстрирует:

#### Генерация трёх типов диаграмм:

- UML Sequence (диаграммы последовательности);

- BPMN (бизнес-процессы);

- Маркетинговая воронка (funnel).

#### Чат-интерфейс (правый сайдбар), в который пользователь вводит запрос на русском или английском (например: «Покажи последовательность оплаты заказа с проверкой платежа» или «Нарисуй воронку продаж: просмотр → корзина → оплата → доставка»).

#### Мультиагентная обработка запроса:

1) Оркестратор определяет тип диаграммы.
2) Агент структурирования преобразует текст в JSON-описание элементов (валидируется Pydantic-схемой).
3) Агент валидации проверяет логику (наличие начальных/конечных элементов, корректность связей).
4) Агент визуализации рисует диаграмму с помощью специализированных библиотек:
- BPMN → bpmn.io;
- UML Sequence → jointjs (кастомизированный рендерер);
- Воронка → самописный SVG-рендерер (или создать свой движок).
#### Интерактивный редактор (левый сайдбар) на базе bpmn.io и jointjs, позволяющий:

- Перетаскивать элементы;

- Изменять их свойства (текст, цвет, тип);

- Сохранять изменения обратно в JSON и синхронизировать с чатом;

- Возможность уточнения через чат после генерации: например, «Сделай этот блок синим», «Добавь шаг "проверка документа" после оплаты»;

- Экспорт в PNG, JPG, PDF.

#### Сквозное состояние (state/memory) диалога: система помнит предыдущие запросы и правки в рамках сессии.
